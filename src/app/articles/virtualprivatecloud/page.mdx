## What is a Virtual Private Cloud (VPC)?

A Virtual Private Cloud (VPC) is a secure, individual, private compute model hosted within a public cloud. It allows organizations to create an isolated network environment within Google Cloud to securely connect and manage their resources.

Think of it as a private section of the cloud where you control the network, security, and connectivity.

## Why Use a VPC in Google Cloud?

Google Cloud VPCs provide several advantages:

- ‚úÖ Security & Isolation ‚Äì Keep your resources private while defining strict access controls.
- ‚úÖ Scalability ‚Äì Easily scale networking across regions and zones.
- ‚úÖ Flexibility ‚Äì Choose between using a default VPC or creating a custom VPC tailored to your needs.
- ‚úÖ Global Reach ‚Äì Unlike traditional on-premise networks, Google Cloud VPCs are global by default, enabling seamless communication across resources.

1. Creating a VPC Using Google Cloud CLI (gcloud)

Run the following command in your terminal to create a custom VPC in Google Cloud:

```bash
gcloud compute networks create my-custom-vpc \
    --subnet-mode=custom \
    --bgp-routing-mode=global
```

This will create a VPC named my-custom-vpc with custom subnet mode (so you define your own subnets instead of using Google‚Äôs default settings).

Now, let‚Äôs create a subnet inside this VPC:

```bash
gcloud compute networks subnets create my-subnet \
    --network=my-custom-vpc \
    --range=10.0.0.0/24 \
    --region=asia-east1
```

Here‚Äôs what happens:
- ‚úî Subnet Name: my-subnet
- ‚úî VPC: my-custom-vpc
- ‚úî IP Range: 10.0.0.0/24
- ‚úî Region: asia-east1

Now, if you want to allow SSH access, create a firewall rule:
```bash
gcloud compute firewall-rules create allow-ssh \
    --network=my-custom-vpc \
    --allow=tcp:22
```

2. Creating a VPC Using Terraform

Terraform allows you to manage infrastructure as code. Here‚Äôs how you can define a VPC in Terraform:

```hcl

  name                    = "my-custom-vpc"
  auto_create_subnetworks = false
  routing_mode            = "GLOBAL"
}
```

Now, define a subnet inside this VPC:

```hcl

  name          = "my-subnet"
  network       = google_compute_network.vpc_network.id
  ip_cidr_range = "10.0.0.0/24"
  region        = "asia-east1"
}
```

And if you want to allow SSH access, define a firewall rule:

```hcl
resource "google_compute_firewall" "allow_ssh" {
  name    = "allow-ssh"
  network = google_compute_network.vpc_network.id

  allow {
    protocol = "tcp"
    ports    = ["22"]
  }
}
```

Apply the Terraform Configuration

After writing this Terraform file (main.tf), run the following commands:

```bash
terraform init
terraform apply -auto-approve
```
